비트 반전 배열(rev) 계산

현재는 매번 ntt() 호출 시 rev[]를 새로 계산합니다.

입력 크기 n이 고정되거나 자주 반복된다면, rev[]를 미리 계산해 캐싱하면 불필요한 반복을 줄일 수 있습니다.

루프 내 모듈러 연산

if(x>mod)x-=mod; if(y<0)y+=mod; 같은 조건문은 분기 예측 실패 시 성능 저하를 유발합니다.

x -= (x >= mod) * mod; y += (y >> 63) & mod; 같은 분기 없는 방식으로 바꾸면 CPU 파이프라인 효율이 좋아집니다.

powmod 호출 최적화

ang = powmod(w, (mod - 1) / n);는 매번 호출됩니다.

자주 쓰이는 크기 n에 대해 미리 원시근을 계산해 캐싱하면 속도가 빨라집니다.

메모리 접근 최적화

a[i] = (i < na) ? _a[i] : 0 같은 초기화는 루프마다 조건 분기를 포함합니다.

memcpy + memset을 활용해 한 번에 복사/초기화하면 더 빠릅니다.

carry 처리

현재는 for (i = 0; i < nc; ++i)에서 자리올림을 순차적으로 처리합니다.

큰 수 곱셈에서는 병렬화가 어렵지만, __int128을 활용해 더 큰 단위로 처리하거나 루프 언롤링을 적용하면 성능 향상 가능.

출력 변환 최적화

tmp[]를 이용해 매번 자리수를 역순으로 저장 후 다시 출력하는 방식은 비용이 큽니다.

sprintf 또는 직접 버퍼에 쓰는 방식을 개선하면 출력 속도가 빨라집니다.
